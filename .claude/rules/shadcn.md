# ShadCN UI Rules

- Use ShadCN UI components for all common UI patterns and accessibility needs
- Follow the ShadCN UI documentation and best practices for component usage and customization
- Ensure that all components are styled consistently with the project's design system and Tailwind CSS configuration
- When creating custom components, extend or compose ShadCN UI components where possible to maintain consistency
- Avoid reinventing common UI patterns that are already provided by ShadCN UI, such as buttons, forms, modals, and navigation elements
- Regularly review the ShadCN UI library for updates and new components that can be adopted into the project to improve functionality and user experience
